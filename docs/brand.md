# Pitot identity v1

## Positioning

- Name: Pitot
- Descriptor: Open sensor and control transport for coding-agent tooling
- Primary line: **Pitot reports. Your controller decides.**
- Voice: exact, calm, physical, technically honest

## Mark

The paired horizontal channels represent total- and static-pressure readings.
They stay separate until the vertical datum resolves them into one downstream
signal. The orange pickups receive; they do not point, command, or block.

## Colour

| Token | Value | Source |
|---|---:|---|
| Manual stock | `#EEE9DC` | Warm technical documentation paper |
| Paper | `#F8F5EC` | README and diagram field |
| Instrument ink | `#182224` | Cockpit/instrument legibility |
| Secondary ink | `#394446` | Supporting copy |
| Calibration rule | `#A9AAA2` | Drafting and measurement lines |
| Probe-cover orange | `#E4572E` | Physical remove-before-flight visibility |

## Typography

- README body: GitHub system typography
- Diagram display: Trebuchet MS / Segoe UI / sans-serif
- Technical labels: IBM Plex Mono / SFMono-Regular / Consolas / monospace
- Never use orange for long copy; it marks inputs, correlation, and selected signal.

## Prohibited forms

- aircraft silhouettes, wings, orbit lines, stars, NASA-like lettering;
- robots, brains, sparkles, chat bubbles, shields, or locks;
- cyan/purple neon gradients;
- arrows implying Pitot makes the decision; and
- combining the Pitot mark with a controller badge in a way that erases the
  capability boundary.
